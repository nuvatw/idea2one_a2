# 三、背景簡介

## 3.1 目標

建立一款 **簡單、可快速開始、可打發零碎時間的貪吃蛇遊戲**，讓使用者可以在 Web 瀏覽器中快速遊玩。

核心目標：

* 開啟頁面即可遊玩
* 遊戲邏輯清晰
* UI 簡潔
* 可短時間重複遊玩

---

## 3.2 背景

貪吃蛇是一個經典街機遊戲，玩法簡單但具有可重複性，非常適合：

* 等待時間
* 休息時間
* 打發碎片時間

此產品的定位為：

> 一個 **零門檻、無登入、可立即開始遊玩的輕量 Web 小遊戲**

---

## 3.3 產品價值

對使用者的價值：

* 快速娛樂
* 操作簡單
* 成就感（分數）

對開發的價值：

* 建立完整小遊戲產品流程
* 可延伸更多遊戲機制

---

# 四、產品範圍

## 4.1 本次範圍（MVP）

本版本將包含：

1. 貪吃蛇核心遊戲
2. 分數系統
3. 遊戲開始 / 結束流程
4. 重玩機制
5. 基本 UI
6. 基本動畫
7. 鍵盤操作

---

## 4.2 本次不做（Out of Scope）

以下功能 **不在 MVP 範圍**：

* 使用者登入
* 排行榜
* 雲端分數保存
* 多人模式
* 遊戲皮膚
* 地圖模式
* 關卡模式
* 音效與背景音樂
* 道具系統

未來版本可能加入：

* 全球排行榜
* 手機觸控操作
* 遊戲難度
* 遊戲皮膚

# 七、功能流程

## 7.1 遊戲流程

```
進入頁面
   ↓
顯示開始畫面
   ↓
點擊 Start
   ↓
遊戲開始
   ↓
蛇移動
   ↓
吃到食物？
   ├─ 是 → 分數+1 → 蛇變長
   └─ 否 → 繼續移動
   ↓
撞牆 / 撞自己？
   ├─ 是 → Game Over
   └─ 否 → 繼續遊戲
   ↓
顯示分數
   ↓
玩家選擇 Restart
```

隨機生成：

```
food = {x:5,y:7}
```

限制：

* 不可生成於蛇身上

---

## 10.3 分數

```
score = 0
```

# 十一、成功指標

由於為輕量遊戲，成功指標主要為使用體驗。

核心指標：

| 指標     | 目標     |
| ------ | ------ |
| 遊戲可啟動率 | 100%   |
| 遊戲可完成率 | 100%   |
| 操作延遲   | < 50ms |
| 平均遊玩時間 | > 2 分鐘 |

---

# 十二、驗收標準

## 12.1 遊戲啟動

條件：

* 開啟頁面
* 點擊 Start

驗收：

* 遊戲開始
* 蛇開始移動

---

## 12.2 吃到食物

條件：

* 蛇頭與食物座標一致

驗收：

* 分數 +1
* 蛇長度 +1
* 食物重新生成

---

## 12.3 碰撞牆壁

條件：

蛇頭超出邊界

驗收：

* 遊戲停止
* 顯示 Game Over

---

## 12.4 碰撞自身

條件：

蛇頭碰到身體

驗收：

* Game Over

---

## 12.5 Restart

條件：

Game Over 畫面點擊 Restart

驗收：

* 分數歸零
* 蛇回到初始長度
* 重新開始

---

# 十三、Corner Case

## 13.1 食物生成在蛇身上

處理方式：

* 重新生成食物位置

---

## 13.2 玩家快速反向操作

例如：

```
Right → Left
```

處理：

* 不允許 180° 反向

---

## 13.3 同時按兩個方向鍵

處理：

* 以最後輸入為準

---

## 13.4 食物生成在邊界

允許。

---

## 13.5 蛇佔滿地圖

處理：

* 顯示 Victory（可選）
* 或 Game Over

（MVP 可忽略）
